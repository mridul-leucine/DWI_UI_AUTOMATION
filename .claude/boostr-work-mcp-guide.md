# Boostr Work MCP Usage Guide

## Workspace ID
Always ask the user for the workspace ID if not provided. Never assume it.

## Answering Codebase Questions
When users ask about code ("How does X work?", "What will change if I modify X?", bug fixes, new features):
- Use the **answer_question** tool from Boostr Work MCP
- Provides comprehensive answers following existing code patterns
- Ensures you don't miss critical context

## Development Plans

**Getting Plans**: Always ask for the dev plan ID if not provided.

**Task Management**:
- Mark tasks completed using MCP after finishing them
- Retrieve task details from MCP if needed
- **Important**: Save the dev plan to your todo list immediately after fetching, before starting work
